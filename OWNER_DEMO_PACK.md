# nutri-SCULPT Dashboard Owner Demo Pack

## Copy-Paste Message To Send

Hi [Owner name],

I have been working on a simple mobile-friendly companion app for the nutri-SCULPT 8 Week Challenge.

The idea is to make the PDF easier for clients to follow day to day. Instead of paging through the e-book, the client can open one link and:

- plan meals for the week
- choose meals from the PDF meal bank
- add custom meals, products and ingredients
- track daily habits
- track workouts and progress
- build a shopping list automatically
- use a simple shopping mode in store
- check calories/macros against their daily targets

Demo link:
https://yolandeschulz.github.io/nutri-sculpt-dashboard/index.html?v=20260708-29

It is currently a prototype/demo version. It is open so you can test it easily without logging in. For a commercial rollout, we can add paid member access, user accounts, Paystack subscriptions, and private data storage.

A few important notes:

- Data currently saves on the user's own phone/browser.
- The app is not yet connected to paid subscriptions.
- The app should be reviewed against the official programme wording before client launch.
- Any commercial use should confirm that the PDF/programme content can be used in this digital format.
- The app is general wellness support and should not replace medical or dietitian advice.

I would love your feedback on whether this could help clients follow the challenge more easily, and whether it is worth turning into a paid member app for all clients.

## What To Ask The Owner To Test

Please test mostly on a phone, because that is where clients will use it.

### First Impression

- Does the opening screen make sense?
- Is it clear what the client should do today?
- Is the wording simple enough for non-tech-savvy clients?
- Does it feel like a helpful companion rather than a complicated tracker?

### Meal Planning

- Go to My Week.
- Choose meals for 3 days.
- Check if the chosen meals are easy to understand.
- Clear or change one meal.
- Check if Sunday feels flexible and not like a failure.

### Today View

- Go to Today.
- Check if the meals for the selected day appear clearly.
- Check if calories and protein are easy to see.
- Tick a few daily habits.
- Check if the daily habit percentage makes sense.

### Custom Foods And Meals

- Go to Create Meal.
- Add a packaged product.
- Add an ingredient.
- Build a simple custom meal.
- Check if saved products/ingredients are easy to reuse.

### Shopping List

- Go to Shopping List.
- Change between Today, Next 3 days and Full week.
- Mark a few items as Need to buy, Already have, Optional and Skip.
- Start shopping mode.
- Tick an item as bought.
- Check if this feels useful in a real shop.

### Workouts And Progress

- Go to Workouts.
- Check if 3 weekly workouts are clear.
- Mark one as done.
- Go to Progress.
- Add a sample measurement or reflection.
- Check if the wording feels supportive.

### Learn Section

- Go to Learn.
- Check if the rules are readable.
- Check if it feels easier than searching through the PDF.

## Feedback Questions For The Owner

1. Would this make the 8-week challenge easier for clients to follow?
2. Which screen would clients use most often?
3. What feels confusing or unnecessary?
4. What must be changed before real clients use it?
5. Should clients be able to edit targets, meals and custom foods?
6. Should coaches/admins see client progress, or should it stay private to each client?
7. Would this work as part of the existing challenge price, or as a separate subscription?
8. If subscription-based, is R99/month reasonable?
9. Should payment be monthly, once-off per challenge, or included in the programme?
10. Does the app need branding changes before launch?

## Current Demo Limitations

- It is currently open to anyone with the link.
- User data is stored on each device, not in a central coach dashboard.
- Paystack subscriptions are not connected yet.
- Login/member access is built but switched off for demo sharing.
- Nutrition label/photo reading is still a helper tool and may need more refinement.
- The app is a PWA/browser app, not yet an App Store or Play Store native app.

## Commercial Rollout Checklist

Before using this with paying clients:

- Confirm permission to use the programme/PDF content digitally.
- Add terms and privacy policy.
- Add clear medical/wellness disclaimer.
- Turn login/member access back on.
- Connect Paystack subscription payments.
- Decide whether each client keeps private local data or has cloud backup.
- Decide whether coaches need an admin dashboard.
- Add a custom domain.
- Run mobile testing on older Samsung phones and iPhones.
- Review all meal/rule wording with the programme owner.
